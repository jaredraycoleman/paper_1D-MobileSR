There are two policemen, one with speed 1 and the other with speed u < 1. The policemen are lazy and were chatting with each other.
By the time they get the alert that a fugitive has escaped, they are already at an unknown distance d from the jail (and may have passed them).
Their initial location is distance s from the jail.

The target is moving away from the jail at speed v < 1.

One option is for the faster policeman to just execute the single-agent algorithms by herself. Another algorithm, which we call the split-up algorithm, is for the faster agent to move toward the jail (origin) and the slow agent away from the jail. If the faster agent reaches the jail and does not find the fugitive, then it must be that d > s and so it starts moving away from the jail. In this case, there are two cases that may occur:
1. The faster agent may pass the slower agent before it finds the fugitive. In this case the competitive ratio can be written:
Opt:=(2(d-s)/(1-v)+s) (*In the offline alg, the fast agent goes straight to the target and back to the jail*)
CaptureTimeCase1:=(d+s)/(1-v) (*Time when fast agent catches up to escaping fugitive without slow agent's help*)
CRCase1=FullSimplify[(2CaptureTimeCase1-s)/Opt]
Out[45]= -1-(4 d)/(-2 d+s+s v)
2. The slower agent finds the target first and so turns around to start moving back to the jail with it. Then, the faster agent will encounter the slower agent with the fugitive, can pick him up and finish the delivery. In this case the competitive ratio can be written:
CaptureTimeCase2:=(d-s)/(u-v) (*Time slow agent catches up to fugitive*)
CaptureLocationCase2:=s+CaptureTimeCase2/u  (*Location (distance from 0) that slow agent catches up to fugitive*)
MeetingTimeCase2:=((CaptureLocationCase2-s)+(2s) )/(1+u)
CRCase2=FullSimplify[(2*MeetingTimeCase2-s)/Opt]
Out[49]= ((-1+v) (-s+(2 (2 s+(d-s)/(u^2-u v)))/(1+u)))/(-2 d+s+s v)
Notice that case 2 can only occur when u > v and when CaptureTimeCase2 < CaptureTimeCase1.
